# Personal Website Starter

This is a beginner-friendly Jekyll starter for a personal aerospace engineering website. It is set up so you can write most of the content in Markdown and keep your personal details in one file.

## Files you will edit most often

- `_data/profile.yml`: your name, bio, email, social links, CV file path, homepage highlights
- `index.md`: short homepage introduction
- `about.md`: longer personal background
- `projects.md`: project portfolio content
- `contact.md`: contact details page
- `assets/files/cv.pdf`: replace the placeholder with your real CV PDF

## How to run the site locally on Windows

1. Install Ruby for Windows.
2. Open PowerShell in this folder.
3. Run `bundle install`.
4. Run `bundle exec jekyll serve`.
5. Open the local address shown in the terminal, usually `http://127.0.0.1:4000/`.

## If you want to host it with GitHub Pages later

1. Push this folder to a GitHub repository.
2. Enable GitHub Pages in the repository settings.
3. If you want, I can also convert this starter into a GitHub Pages specific setup.

## How to edit this site on GitHub

If you want to make quick edits in the GitHub website:

1. Open the file in your repository.
2. Click the pencil icon to edit it.
3. Make your changes.
4. Commit directly to `main`.

The files you will usually edit are:

- `_data/profile.yml`: name, headline, email, LinkedIn, homepage facts, profile image path
- `index.md`: homepage intro text
- `about.md`: about page text
- `projects.md`: projects page text
- `contact.md`: contact page details

To change your profile picture:

1. Upload your image to `assets/images/`
2. Open `_data/profile.yml`
3. Change `profile_image` to the new file path, for example `/assets/images/nduka-photo.jpg`

To replace your CV:

1. Upload the new PDF to `assets/files/cv.pdf`
2. Commit the change and GitHub Pages will update automatically
